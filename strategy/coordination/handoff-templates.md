# 📋 NEXUS 交接模板

> NEXUS 流水线中每种类型 Agent 间交接的标准化模板。一致的交接可防止上下文丢失 —— 这是多 Agent 协调失败的第一大原因。

---

## 1. 标准交接模板

用于任何 Agent 间工作转移。

```markdown
# NEXUS 交接文档

## 元数据
| 字段 | 值 |
|------|-----|
| **来源** | [Agent 名称]（[部门]） |
| **目标** | [Agent 名称]（[部门]） |
| **阶段** | Phase [N] —— [阶段名称] |
| **任务引用** | [Sprint Prioritizer backlog 中的任务 ID] |
| **优先级** | [关键 / 高 / 中 / 低] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 上下文
**项目**：[项目名称]
**当前状态**：[到目前为止已完成的工作 —— 要具体]
**相关文件**：
- [文件/路径/1] —— [包含什么内容]
- [文件/路径/2] —— [包含什么内容]
**依赖**：[此工作依赖什么完成]
**约束**：[技术、时间线或资源约束]

## 交付物请求
**需要什么**：[具体、可衡量的交付物描述]
**验收标准**：
- [ ] [标准 1 —— 可衡量]
- [ ] [标准 2 —— 可衡量]
- [ ] [标准 3 —— 可衡量]
**参考材料**：[链接到规格、设计、之前的工作]

## 质量期望
**必须通过**：[此交付物的具体质量标准]
**所需证据**：[完成证明的样子]
**交接给下一步**：[谁接收输出以及他们需要什么格式]
```

## 2. QA 反馈循环 —— PASS

当 Evidence Collector 或其他 QA Agent 批准任务时使用。

```markdown
# NEXUS QA 裁决：PASS ✅

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者 Agent** | [Agent 名称] |
| **QA Agent** | [Agent 名称] |
| **尝试** | [N] / 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：PASS

## 证据
**截图**：
- 桌面（1920x1080）：[文件名/路径]
- 平板（768x1024）：[文件名/路径]
- 移动端（375x667）：[文件名/路径]

**功能验证**：
- [x] [验收标准 1] —— 已验证
- [x] [验收标准 2] —— 已验证
- [x] [验收标准 3] —— 已验证

**品牌一致性**：已验证 —— 颜色、排版、间距匹配设计系统
**可访问性**：已验证 —— 键盘导航、对比度、语义化 HTML
**性能**：[测量的加载时间] —— 在可接受范围内

## 备注
[任何观察、未来改进的小建议或正面反馈]

## 下一步行动
→ Agents Orchestrator：标记任务完成，推进到 Backlog 中的下一个任务
```

## 3. QA 反馈循环 —— FAIL

当 Evidence Collector 或其他 QA Agent 拒绝任务时使用。

```markdown
# NEXUS QA 裁决：FAIL ❌

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者 Agent** | [Agent 名称] |
| **QA Agent** | [Agent 名称] |
| **尝试** | [N] / 3 |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 裁决：FAIL

## 发现的问题

### 问题 1：[类别] —— [严重性：关键/高/中/低]
**描述**：[问题的精确描述]
**预期**：[根据验收标准应该发生什么]
**实际**：[实际发生了什么]
**证据**：[截图文件名或测试输出]
**修复指令**：[具体、可操作的解决指令]
**要修改的文件**：[精确的文件路径]

### 问题 2：[类别] —— [严重性]
**描述**：[...]
**预期**：[...]
**实际**：[...]
**证据**：[...]
**修复指令**：[...]
**要修改的文件**：[...]

[继续列出所有发现的问题]

## 验收标准状态
- [x] [标准 1] —— 通过
- [ ] [标准 2] —— 失败（见问题 1）
- [ ] [标准 3] —— 失败（见问题 2）

## 重试指令
**给开发者 Agent**：
1. 只修复上面列出的问题
2. 不要引入新功能或更改
3. 所有问题解决后重新提交 QA
4. 这是尝试 [N] / 最多 3 次

**如果第 3 次尝试失败**：任务将升级到 Agents Orchestrator
```

## 4. 升级报告

当任务超过 3 次重试尝试时使用。

```markdown
# NEXUS 升级报告 🚨

## 任务
| 字段 | 值 |
|------|-----|
| **任务 ID** | [ID] |
| **任务描述** | [描述] |
| **开发者 Agent** | [Agent 名称] |
| **QA Agent** | [Agent 名称] |
| **已耗尽尝试** | 3/3 |
| **升级给** | [Agents Orchestrator / Studio Producer] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 失败历史

### 尝试 1
- **发现的问题**：[摘要]
- **应用的修复**：[开发者改变了什么]
- **结果**：FAIL —— [为什么仍然失败]

### 尝试 2
- **发现的问题**：[摘要]
- **应用的修复**：[开发者改变了什么]
- **结果**：FAIL —— [为什么仍然失败]

### 尝试 3
- **发现的问题**：[摘要]
- **应用的修复**：[开发者改变了什么]
- **结果**：FAIL —— [为什么仍然失败]

## 根本原因分析
**为什么任务一直失败**：[潜在问题的分析]
**系统性问题**：[这是偶发还是模式？]
**复杂性评估**：[任务是否被正确界定？]

## 推荐解决方案
- [ ] **重新分配**给不同的开发者 Agent（[推荐的 Agent]）
- [ ] **分解**为更小的子任务（[建议的分解]）
- [ ] **修订方法** —— 需要架构/设计变更
- [ ] **接受**当前状态并记录已知限制
- [ ] **推迟**到未来冲刺

## 影响评估
**阻塞**：[什么其他任务被此阻塞]
**时间线影响**：[这如何影响整体进度]
**质量影响**：[如果我们接受当前状态存在什么质量妥协]

## 需要决策
**决策者**：[Agents Orchestrator / Studio Producer]
**截止时间**：[何时需要决策以避免进一步延误]
```

## 5. 阶段门控交接

在 NEXUS 阶段之间转换时使用。

```markdown
# NEXUS 阶段门控交接

## 转换
| 字段 | 值 |

| **来源阶段** | Phase [N] —— [名称] |
| **目标阶段** | Phase [N+1] —— [名称] |
| **门控负责人** | [Agent 名称] |
| **门控结果** | [PASSED / FAILED] |
| **时间戳** | [YYYY-MM-DDTHH:MM:SSZ] |

## 门控标准结果
| # | 标准 | 阈值 | 结果 | 证据 |
|---|------|------|------|------|
| 1 | [标准] | [阈值] | ✅ PASS / ❌ FAIL | [证据引用] |
| 2 | [标准] | [阈值] | ✅ PASS / ❌ FAIL | [证据引用] |
| 3 | [标准] | [阈值] | ✅ PASS / ❌ FAIL | [证据引用] |

## 转交的文档
1. [文档名称] —— [下一阶段的用途]
2. [文档名称] —— [下一阶段的用途]

## 下一阶段的关键约束

## 下一阶段的 Agent 激活

## 转交的风险

```

## 7. 事件交接

在事件响应期间使用。

```markdown
# NEXUS 事件交接

## 事件
| 字段 | 值 |
|------|-----|
| **严重性** | [P0 / P1 / P2 / P3] |
| **发现者** | [Agent 或系统] |
| **发现时间** | [时间戳] |
| **分配给** | [Agent 名称] |
| **状态** | [调查中 / 缓解中 / 已解决 / 事后分析] |

## 描述
**发生了什么**：[事件的清晰描述]
**影响**：[谁/什么受到影响以及严重程度]
**时间线**：
- [HH:MM] —— [事件]
- [HH:MM] —— [事件]
- [HH:MM] —— [事件]

## 当前状态
**受影响的系统**：[列表]
**是否有变通方案**：[是/否 —— 如是请描述]
**预计解决时间**：[时间估算]

## 已采取的行动
1. [已采取的行动及结果]
2. [已采取的行动及结果]

## 交接上下文
**给下一响应者**：
- [已尝试过什么]
- [还没尝试过什么]
- [怀疑的根本原因]
- [要检查的相关日志/指标]

## 利益相关者沟通
**上次更新发送**：[时间戳]
**下次更新到期**：[时间戳]
**沟通渠道**：[更新发布在哪里]
```

## 使用指南

| 情况 | 使用的模板 |
|------|-----------|
| 向其他 Agent 分配工作 | 标准交接（#1） |
| QA 批准任务 | QA PASS（#2） |
| QA 拒绝任务 | QA FAIL（#3） |
| 任务超过 3 次重试 | 升级报告（#4） |
| 在阶段之间移动 | 阶段门控交接（#5） |
| 冲刺结束 | 冲刺交接（#6） |
| 系统事件 | 事件交接（#7） |
